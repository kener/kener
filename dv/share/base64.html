数据URI模式

数据URI模式 (Uniform Resource Identifier scheme，统一资源识别模式) 提供了一种网页内嵌数据（如外部数据）的方式. 这种技术允许通过单次HTTP而不是多次请求获得所有图片、CSS等资源。在某些情况下，这种单文档资源获取方式更加高效。
在浏览器端使用Data URI能够提供不依赖服务端的下载行为，比如，使用JS生成的图片或者数据要给用户下载， 那么提供一个地址给location.href触发下载 'data:text/csv;charset=UTF-8,' + encodeURIComponent(csv) IETF在1998年就发布了相关的data URI 协议标准, 而且HTML4.0+就开始支持 the data URI， 在大部分浏览器中都支持该协议。 数据URI的基础就是将资源统一用base64的形式编码（包括文本和二进制文件），base64数据的头部信息说明了资源的信息（类型，编码，格式）

文件下载 —— 以文件之名下载

客户端直接下载

以前，我们下载一个资源需要向服务端发送一起请求，而且只能通过浏览器输入地址，或者点击锚链接； HTML5为离线应用为A增加了一个新特性，能够让资源按指定的文件名下载，为动态资源起个好听的名字是个好注意。 配合base64，我们能够让前端产生的内容可下载，而且可指定名称下载；当然第一步是让资源base64编码，然后让A.href引用，download属性指定下载的名称. 如果要通过JS触发下载行为，采用模拟点击行为吧，放心，下载行为不会刷新你的页面； 嗯，存在的问题，IEs目前不支持 <a href="data:image/png;base64,****" download="sense.png" >sense</a> sense.png 另外，对于离线应用，离线资源和离线数据可以在浏览器本地数据库或本地缓存中，如果用户需要下载这些数据用这种方式下载也是相当方便的，不需要任何服务交互就能实现数据存取。

客户端请求下载，按指定文件名下载

浏览器在下载过程中会检查http头部信息，如果指定了文件名，则会按该文件名下载命名，否则会从URL路径中解析文件名。 header('Content-Disposition: attachment; filename="filedown.zip"'); 如果要用中文命名，为了避免下载的文件是乱码形式最好使用编码方式指定下载文件名， 但是对于不同的浏览器，使用的编码格式不一样。看代码 $filename = $_GET['filename']; if (isset($_SERVER['HTTP_USER_AGENT'])) { $age = strtolower($_SERVER['HTTP_USER_AGENT']); if (strstr($age, 'msie' )){ $filename = urlencode( $_GET['filename'] ); }else if(strstr($age, 'chrome' ) || strstr($age, 'firefox' )){ $filename = base64_encode( $_GET['filename'] ); $filename = "=?UTF8?B?$filename?="; } } //发送文件头部信息，指定文件名 header('Content-Disposition: attachment; filename="' . $filename . '"'); //读取文件内容，并发送到客户端 $content = file_get_content( $filename ); echo $content;
